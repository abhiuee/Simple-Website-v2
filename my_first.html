Lesson 1: Basics of Web and HTML

World Wide Web

The World Wide Web (www) is an open source information space where documents and other web resources are identified by URLs (links), interlinked by hypertext links, and can be accessed via the Internet. Basic elements of web are as follows:
  • Computer
  • Internet
  • Server
  • HTTP Protocol
When a person goes to a web page like www.google.com, their computer sends a HTTP request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen.

HTML

HyperText Markup Language, commonly referred to as HTML, is the standard markup language used to create web pages. Click Here to learn about HTML. A typical HTML document consists of:
  • Text Content
  • Markup
  • References to other documents
  • Links to other pages

Markup

HTML markup consists of several key components, including those called tags (and their attributes), character-based data types, character references and entity references. HTML tags most commonly come in pairs like <b> and </h1>, although some represent empty elements and so are unpaired, for example <img>.
Elements

HTML documents imply a structure of nested elements. A typical element consists of opening tag, text content and closing tag, for example:

<em> hello </em>
HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.
Tag Attributes
Some tags like <img> require modifiers to describe their functionality. These modifiers are called tag attributes or just attributes. An attribute either modifies the default functionality of an element type or provides functionality to certain element types unable to function correctly without them. Example:
<img src="source of image" alt="text">

HTML Document Example

<!DOCTYPE HTML>
<html>
<head>
    <title>
        Title
    </title>
</head>
<body>
        Text Contents
</body>
</html>

Lesson 2: Structured HTML Pages and CSS

CSS

Cascading Style Sheet (CSS) is a style sheet language used for describing the presentation of a document written in a markup language. CSS is designed primarily to enable the separation of document content from document presentation, including aspects such as the layout, colors, and fonts. For more information about CSS, please click here.

DOM

The Document Object Model (DOM) is a cross-platform and language-independent convention for representing and interacting with objects in HTML, XHTML, and XML documents. The nodes of every document are organized in a tree structure, called the DOM tree. Objects in the DOM tree may be addressed and manipulated by using methods on the objects.
The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on... In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

Lesson 3: CSS Styling Concepts

Box Sizing

HTML elements are boxes and each box has the following four components:
  • Margin
  • Border
  • Padding
  • Content
To deal with sizing issues:
  • Set size in terms of percentage
  • Set box-sizing attribute to border-box for every element

Box Positioning

div is a block element (as opposed to inline element), so they take entire width of page by default. Adding the rule display:flex to the CSS will override the behavior and let divs appear next to each other.

Developer Tools

Developer tools allows web designers and developers to take a look at the make up of their pages. They are usually built into the browser and do not require additional modules or configuration. The tools are used to see how different boxes are arranged in the html page and also help in making minor adjustments to the html page.

Web Development

Servers

A server is a computer program or a machine that waits for requests from other machines or software (clients) and responds to them.[a] The purpose of a server is to share data or hardware and software resources among clients.

How Web Works

When we enter a URL http://wwww.example.com/abc/a.html in the browser, first, the browser slices the URL in 3 parts:
  1. http://: This part indicates that the document you want to access can be retrieved from web server, which understands the HTTP protocol.
  2. www.example.com: This is the host name of the computer from which the document can be downloaded.
  3. /abc/a.html: This is the virtual path of the document in the www.example.com's web server.
Then, the browser contacts a DNS (Domain Name Server) to know the IP address of the computer whose full qualified domain name is www.example.com. The browser establishes a connection channel with the web server on the computer which IP address was given by the DNS server and requests the document using a GET request on the host which name is www.example.com and which virtual path is /abc/a.html. The server decodes the request and maps the virtual path to a real one, which should match an existing file. The server sends the file to the browser with some useful information such as its last modification time and its MIME type. The MIME type helps the browser decide how to display the received document. In our example, it is a HTML file. So the server sets its MIME type to text/html and the browser understands that it must render it as text.

GET and POST

POST is one of many request methods supported by the HTTP protocol used by the web. By design, the POST request method requests that a web server accepts and stores the data enclosed in the body of the request message. It is often used when uploading a file or submitting a completed web form.
The HTTP GET request method is designed to retrieve information from the server. As part of a GET request, some data can be passed within the URL's query string, specifying for example search terms, date ranges, or other information that defines the query.

Differences between GET and POST

GET POST
Query string is sent in the URL Query string is sent in the HTTP message body of the request
Can have length restrictions Usually don't have length restrictions
Can be cached and bookmarked Can never be cached and bookmarked
Should not be used when sending passwords or other sensitive information Relatively more secure and can be used for sending passwords

Web Applications

A web application is a software application that runs on a remote server. In most cases, web browsers are used to access web applications, over a network. Web applications require very little compute power on the client as most of the processing happens on the server and client only displays data. Also, these applications solve some of the compatibility issues.

HTML Validation

HTML Validation is an important concept that allows unscrupulous usage of HTML documents. Escaping is one such feature where any input from users is first validated by removing all the escape characters so that the text is actually interpreted as text rather than some code or HTML tag.

HTML Templates

HTML templates is a feature which allows reuse of HTML by using concepts like variable substitution and inheritance. One such environment that allows HTML templates is Jinja environment which can be used in Google App Engine. Using templates, removes all the bulky HTML from the backend code and all the HTML documents are then generated dynamically.
